Bases: gnn_devtools.graphs.LabeledDirectedGraph

A knowledge graph

Parameters:
  • num_nodes – total number of nodes in the graph
  • num_relations – total number of unique relations in the graph
  • num_attributes – total number of attributes in the graph
  • reflexive – whether nodes can refer to themselves
  • p_relation – the probability for each node of being connected via a relation. If num_relations > 1 then this is expected to be a list with as many values as there are relations.
  • p_attribute – the probability for each node of having one or more attributes. If num_attributes > 1 then this is expected to be a list with as many values as there are attributes.
  • attribute_config – list of dictionaries, one per feature, with parameters specifying how to generate the values. Necessary keys are ‘mode’, ‘embedding_dim’, ‘distribution’, and ‘p’. If the list is empty, then num_attributes configurations are generated randomly.
  • node_embeddings – use one-hot encoded or random embeddings
  • node_embedding_dim – size (width) of embeddings if random
  • seed – seed for random number generator
